# 项目协作与开发约束

## 当前项目

SupplyFlow AI｜供应链执行 Copilot

这是一个面向洞隐科技 AI 原生产品经理岗位的岗位定制型 MVP / Concept MVP，用于展示对 B 端供应链 SaaS、AI Copilot、Agent 工作流和产品原型落地的理解。

## 技术方向

- Vite + React Web 工作台
- 默认使用 Mock 数据
- 预留 DeepSeek API 接入位置
- 不写死真实 API Key
- 不引入复杂后端、数据库、登录、权限、多租户能力

## 协作规则

- 每个阶段完成后必须停下来，等待用户确认。
- 不要自动进入下一阶段。
- 不要再做自动审核或自动评审。
- 只在用户明确要求时，才执行代码审查、自动检查、重构建议或下一阶段开发。
- 如需运行依赖安装、构建、测试或开发服务器，应先说明目的；遇到权限或网络问题时，不要反复重试。

## 第一阶段边界

第一阶段只完成：

- 项目基础结构
- B 端 SaaS 工作台布局
- 三个示例案例
- 核心 Mock 数据
- 异常输入区和模拟 OMS / WMS / TMS 数据面板

第一阶段不要继续实现：

- AI 诊断面板
- 影响分析
- 处理方案推荐
- 协同话术生成
- 历史记录
- 用户反馈

## 产品边界

不要做：

- 登录系统
- 真实数据库
- 权限系统
- 支付系统
- 复杂多租户 SaaS
- 真实 OMS / WMS / TMS 接口
- 复杂图谱
- 花哨动画
- 普通聊天机器人界面
